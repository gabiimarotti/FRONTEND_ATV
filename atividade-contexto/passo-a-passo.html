Guia de Implementação: React Local (Contexto Clone)

Este guia detalha a arquitetura e os snippets principais para implementar um clone do jogo Contexto utilizando React diretamente no navegador via CDNs, dispensando o uso de Node.js ou bundlers como Vite/Webpack.

1. Setup do Ambiente e CDNs (index.html)

Para rodar React nativamente no navegador, é necessário importar as bibliotecas do React (Core e DOM) e o Babel. O Babel transpilará o JSX para JavaScript puro (ES5) em tempo real de execução (runtime).

No <head> do seu arquivo HTML, adicione as seguintes tags de script:

<!-- Bibliotecas base do React -->
<script src="https://unpkg.com/react@18/umd/react.production.min.js" crossorigin></script>
<script src="https://unpkg.com/react-dom@18/umd/react-dom.production.min.js" crossorigin></script>

<!-- Transpilador Babel Standalone -->
<script src="https://unpkg.com/@babel/standalone/babel.min.js"></script>
Importante: Ao carregar seu script local contendo JSX, você deve especificar o atributo type="text/babel" na tag de importação (ex: <script type="text/babel" src="app.js"></script>).

2. Gerenciamento de Estado (useState)

Como não estamos utilizando um sistema de módulos (ES Modules com import/export), os hooks do React são extraídos diretamente do objeto global React.

O estado (state) é responsável por engatilhar re-renderizações na interface quando mutado.

// Desestruturação do hook useState a partir do objeto global do React
const { useState } = React;

function App() {
    // input state: controla o two-way data binding do campo de texto
    const [inputValue, setInputValue] = useState("");
    
    // Array de objetos armazenando as tentativas ({ word: string, rank: number })
    const [guesses, setGuesses] = useState([]);
    
    // ... resto do componente
}

3. Lógica de Submissão e Ordenação (app.js)

A função manipuladora do formulário é responsável por interceptar o submit, sanitizar o input, buscar o índice da palavra na array do dicionário (distância semântica) e atualizar o estado de forma imutável.

const handleSubmit = (e) => {
    // Previne o reload da página
    e.preventDefault();
    
    const currentGuess = inputValue.trim().toLowerCase();
    if (!currentGuess) return;

    // dictionaryArray é a sua variável global carregada via palavras.js
    // indexOf retorna o índice da palavra (0 é a resposta correta). Retorna -1 se não existir.
    const index = dictionaryArray.indexOf(currentGuess);
    
    if (index === -1) {
        // Implemente sua lógica de erro aqui
        return;
    }

    // O rank real (posição) é o index + 1
    const rank = index + 1;

    // Atualização de estado baseada no estado anterior (Functional State Update)
    setGuesses(prevGuesses => {
        const newGuesses = [...prevGuesses, { word: currentGuess, rank }];
        // Ordena os objetos in-place priorizando o menor rank
        return newGuesses.sort((a, b) => a.rank - b.rank);
    });

    setInputValue("");
};

4. Renderização do JSX e Estilização Dinâmica

No React, a iteração de arrays para renderização de elementos no DOM é feita utilizando o método .map() nativo do JavaScript.

Para aplicar propriedades de CSS dinamicamente com base nos dados (como a largura da barra de proximidade), utiliza-se o atributo style passando um objeto JavaScript.

return (
    <div className="game-container">
        {/* Binding do manipulador ao formulário */}

            <input 
                value={inputValue} 
                onChange={(e) => setInputValue(e.target.value)}

        </form>

        <div className="guesses-list">
            {guesses.map((guess) => {
                // Lógica percentual baseada no rank (1 = 100%)
                const percent = calculatePercentage(guess.rank);
                
                return (
                    <div key={guess.word} style={{
                        background: `linear-gradient(90deg, #2ea043 ${percent}%, transparent ${percent}%)`
                    }}>
                        <span>{guess.word}</span>
                        <span>Posição: {guess.rank}</span>
                    </div>

            })}
        </div>
    </div>
);
Boas Práticas: Note o uso do atributo key dentro do método map. O React utiliza essa chave (que deve ser única para cada elemento irmão) para rastrear o ciclo de vida do elemento no Virtual DOM e otimizar re-renderizações.

5. Renderizando na Raiz (Mounting)

Após definir o componente principal (App), é necessário atrelá-lo ao DOM real do navegador, renderizando-o na <div id="root"> do seu HTML.

// Cria a raiz do React no elemento do DOM
const root = ReactDOM.createRoot(document.getElementById('root'));

// Renderiza o componente na raiz
root.render(<App />);
